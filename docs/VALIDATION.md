# SnapSOAP 검증 계획서 (Validation Plan)

## 1. 핵심 가설 및 비즈니스 임팩트

### 가설 1: 구조화 효율성
> "자유텍스트 진료 기록을 AI로 SOAP 구조화하면,
> 의사의 문서화 시간이 수동 정리 대비 **40% 이상 단축**될 것이다."

- **독립변수**: 구조화 방식 (수동 vs AI 지원)
- **종속변수**: Task Completion Time (초)
- **통제변수**: 진료 기록 길이(100~300자), 복잡도(저/중/고), 참가자 경력

**비즈니스 임팩트**:
- 1차 진료 의사 1인 기준: 일평균 50~100건 진료 × 문서화 2분 = 100~200분/일
- 40% 단축 시 절감 시간: **40~80분/일** → 연 200일 기준 **140~270시간/년**
- 절감 시간을 진료에 재투입 시: 추가 진료 가능 건수 ≈ 10~20건/일 증가
- 클리닉 매출 임팩트 (진료비 1.5만원 기준): 연 **3,000~5,400만 원** 수익 기회 증가
- 문서화 스트레스 감소 → 의사 번아웃 예방 (비재무적 임팩트)

---

### 가설 2: 구조화 정확도
> "AI가 생성한 SOAP 분류 결과의 정확도가 **85% 이상**일 것이다."

- **측정**: 전문가(의사) 리뷰 기반 항목별 정/오 판정
- **기준**: 각 문장이 올바른 SOAP 섹션에 배치되었는지

**비즈니스 임팩트**:
- 정확도 85% 미만 시: 의사가 매번 수정해야 하므로 시간 절감 효과 상쇄
- 85% 이상 시: AI 결과 확인·보완 방식으로 운영 가능 → 신뢰 기반 사용 정착
- 정확도 95% 이상 달성 시: 보험 청구 근거 자료 활용 가능성 → 법적 안전성 확보
- 오분류로 인한 잘못된 처방 위험 없음 (SOAP 구조화는 분류만 수행, 임상 판단 개입 없음)

---

### 가설 3: 사용성
> "SnapSOAP의 SUS(System Usability Scale) 점수가 **70점 이상**일 것이다."

- SUS 70점 = "Good" 등급 (상위 ~30%)
- 목표 사용자(의사, PA, NP) 5명 이상 대상 측정

**비즈니스 임팩트**:
- SUS 70점 미만 시: 자발적 도입 어려움 → 영업·교육 비용 급증
- SUS 70점 이상 시: 클리닉 단위 바이럴 확산 가능 (의사 커뮤니티 추천)
- SUS 80점 이상("Excellent") 달성 시: 기존 EMR 벤더 파트너십 협상 레버리지 확보
- 낮은 SUS는 도입 저항의 직접 지표 → EMR 연동 전 단계에서 반드시 70점 이상 확보 필요

## 2. 성공 지표 (KPI)

| 지표 | 정의 | 목표값 | 측정 방법 | 측정 시점 |
|------|------|--------|-----------|-----------|
| **SOAP 분류 정확도** | AI가 올바른 SOAP 섹션에 배치한 비율 | ≥ 85% | 전문가 2인 독립 리뷰 (10건 × 평균 5항목) | 프로토타입 완성 후 |
| **Task Completion Time** | 입력→구조화→확인 완료까지 소요 시간 | ≤ 수동 대비 60% | 화면 녹화 + 스톱워치 | 사용성 테스트 시 |
| **API 응답 시간** | 구조화 버튼 클릭→결과 표시 | ≤ 5초 (p95) | 서버 로그 `processingTime` 필드 | 실시간 |
| **SUS 점수** | System Usability Scale | ≥ 70점 | SUS 설문 10문항 × Brooke 공식 | 사용성 테스트 후 |
| **Task Completion Rate** | 시나리오 테스트 완료율 | ≥ 90% | 시나리오 관찰 기록지 | 사용성 테스트 시 |
| **수정률** | AI 결과를 사용자가 수정한 항목 비율 | ≤ 20% | 세션 관찰 기록 (수정 발생 항목 수 / 전체 항목 수) | 사용성 테스트 시 |

## 3. 검증 방법

### 3.1 정확도 검증 (Accuracy Test)

**방법**: 10건의 다양한 진료 기록으로 AI 구조화 실행 후,
의료 전문가 2인이 독립적으로 정확도 평가

**평가 기준**:
- ✅ Correct: 문장이 올바른 SOAP 섹션에 배치됨
- ⚠️ Acceptable: 약간 모호하나 합리적 분류 (두 전문가 중 1인 이상 동의)
- ❌ Incorrect: 명백한 오분류 (두 전문가 모두 오류 판정)

**정확도 산출**: (Correct + Acceptable) / Total × 100

**평가자 간 신뢰도**: Cohen's Kappa ≥ 0.7 (실질적 일치) 목표. 미달 시 합의 세션 진행.

**측정 프로토콜**:
1. 10건의 더미 진료 기록 사전 준비 (표 아래 참조)
2. 각 기록을 SnapSOAP에 입력 후 구조화 실행
3. 결과 JSON을 스프레드시트에 항목별 기록
4. 전문가 A, B가 각자 독립적으로 정/오 판정 (의사소통 금지)
5. 불일치 항목에 대해 합의 세션 진행
6. 최종 정확도 산출

**테스트 케이스 구성**:

| # | 진료과 | 복잡도 | 주요 특징 |
|---|--------|--------|-----------|
| 1 | 내과 | 중 | 만성질환 f/u, 검사 결과 다수 |
| 2 | 응급 | 고 | 급성 증상, 빠른 의사결정 |
| 3 | 소아과 | 중 | 보호자 진술, 감염 |
| 4 | 정형외과 | 저 | 단순 외상 |
| 5 | 산부인과 | 중 | 정기 검진 |
| 6 | 내과 | 고 | 다장기 문제, 영어/한국어 혼용 |
| 7 | 정신과 | 중 | 주관적 호소 비중 높음 |
| 8 | 외과 | 중 | 수술 계획 포함 |
| 9 | 내과 | 저 | 간단한 감기 |
| 10 | 응급 | 고 | 다발성 외상 |

### 3.2 사용성 테스트 (Usability Test)

**참가자**: 의사 또는 의료진 3~5명 (모집 가능 범위 내)
**시간**: 참가자당 20~30분
**환경**: Chrome 브라우저, 15인치 이상 모니터, 키보드+마우스

**세션 진행 순서**:
1. 설명 (5분): 프로토타입 고지, Think-aloud 방법 안내, 동의서 서명
2. 태스크 수행 (15분): T1~T5 순서대로 진행, 화면 녹화
3. SUS 설문 (5분): 태스크 완료 직후 10문항 작성
4. 자유 인터뷰 (5분): 불편했던 점, 개선 제안

**시나리오 테스트 (Task-based)**:

| Task | 내용 | 성공 기준 | 제한 시간 |
|------|------|-----------|-----------|
| T1 | SnapSOAP에 접속하여 인터페이스를 파악하세요 | 좌우 패널 역할 설명 가능 | 2분 |
| T2 | 샘플 진료 기록 하나를 선택하여 구조화를 실행하세요 | SOAP 결과 화면 도달 | 1분 |
| T3 | 결과에서 "Assessment" 섹션의 내용을 수정하세요 | 인라인 편집 완료 | 2분 |
| T4 | 직접 진료 기록을 입력하고 구조화하세요 | 본인 작성 텍스트로 구조화 완료 | 5분 |
| T5 | 구조화된 결과를 복사하세요 | 클립보드 복사 Toast 확인 | 1분 |

**측정 항목**:
- 각 태스크 완료 여부 (Success / Fail / Partial)
- 각 태스크 소요 시간 (초, 화면 녹화로 측정)
- 오류 발생 횟수 및 유형 (잘못된 클릭, 기능 탐색 실패 등)
- SUS 설문 (10문항, 홀수 문항: 동의 정도 -1, 짝수 문항: 반전 후 합산 × 2.5)
- Think-aloud 발화 기록

**SUS 산출 공식**:
```
홀수 문항(1,3,5,7,9): 점수 - 1
짝수 문항(2,4,6,8,10): 5 - 점수
합계 × 2.5 = 최종 SUS 점수 (0~100)
```

### 3.3 A/B 비교 (시간 검증)

**방법**: 동일한 진료 기록 3건에 대해 각 방식의 시간 측정

| 조건 | 설명 | 측정 방법 |
|------|------|-----------|
| (A) 수동 정리 | 참가자가 직접 텍스트를 읽고 SOAP 섹션으로 분류. 종이 또는 빈 에디터에 작성 | 스톱워치 (시작: 텍스트 제시, 종료: "완료" 선언) |
| (B) SnapSOAP | AI 구조화 후 결과 확인 및 필요 시 수정 | 스톱워치 (시작: 텍스트 입력 시작, 종료: 결과 확인 완료) |

**측정**: 각 방식의 소요 시간 (초), 최종 결과 품질 (전문가 평가)

**분석**:
- n ≥ 5: 대응 표본 t-test (정규성 가정)
- n < 5: Wilcoxon 부호 순위 검정 (비모수)
- 효과 크기: Cohen's d 산출 (d > 0.5: 중간 효과, d > 0.8: 큰 효과)

## 4. 검증 일정

| 주차 | 활동 | 산출물 |
|------|------|--------|
| Week 1 | 프로토타입 완성 + 내부 테스트 | 버그 리포트, 정확도 예비 측정 (3건) |
| Week 2 | 정확도 테스트 (10건) | 정확도 리포트 (케이스별 판정표, Cohen's Kappa) |
| Week 3 | 사용성 테스트 (3~5명) | SUS 점수, 태스크 완료율, Think-aloud 기록 |
| Week 4 | 결과 분석 + 개선 방향 도출 | 검증 결과 보고서 (가설별 검증 결과 + 다음 액션) |

## 6. 윤리적 고려사항

- 모든 테스트에 **더미 데이터만 사용** (실제 환자 정보 절대 사용 금지)
- 참가자에게 프로토타입임을 명확히 고지 후 서면 동의 획득
- "AI가 의료 판단을 대체하지 않음" 사전 고지 (분류 보조 도구임을 명시)
- 테스트 데이터는 테스트 종료 후 즉시 삭제
- 참가자 발언 및 행동 기록 시 익명 처리 (P1, P2... 코드명 사용)
- 화면 녹화 시 참가자 동의 필수, 연구 목적 외 사용 금지
