# SnapSOAP 검증 계획서 (Validation Plan)

## 1. 핵심 가설

### 가설 1: 구조화 효율성
> "자유텍스트 진료 기록을 AI로 SOAP 구조화하면,
> 의사의 문서화 시간이 수동 정리 대비 **40% 이상 단축**될 것이다."

- **독립변수**: 구조화 방식 (수동 vs AI 지원)
- **종속변수**: Task Completion Time (분)
- **통제변수**: 진료 기록 길이, 복잡도, 참가자 경력

### 가설 2: 구조화 정확도
> "AI가 생성한 SOAP 분류 결과의 정확도가 **85% 이상**일 것이다."

- **측정**: 전문가(의사) 리뷰 기반 항목별 정/오 판정
- **기준**: 각 문장이 올바른 SOAP 섹션에 배치되었는지

### 가설 3: 사용성
> "SnapSOAP의 SUS(System Usability Scale) 점수가 **70점 이상**일 것이다."

- SUS 70점 = "Good" 등급 (상위 ~30%)
- 목표 사용자(의사, PA, NP) 5명 이상 대상 측정

## 2. 성공 지표 (KPI)

| 지표 | 정의 | 목표값 | 측정 방법 | 측정 시점 |
|------|------|--------|-----------|-----------|
| **SOAP 분류 정확도** | AI가 올바른 SOAP 섹션에 배치한 비율 | ≥ 85% | 전문가 리뷰 (10건) | 프로토타입 완성 후 |
| **Task Completion Time** | 입력→구조화→확인 완료까지 소요 시간 | ≤ 수동 대비 60% | 스톱워치 측정 | 사용성 테스트 시 |
| **API 응답 시간** | 구조화 버튼 클릭→결과 표시 | ≤ 5초 (p95) | 서버 로그 | 실시간 |
| **SUS 점수** | System Usability Scale | ≥ 70점 | SUS 설문 10문항 | 사용성 테스트 후 |
| **Task Completion Rate** | 시나리오 테스트 완료율 | ≥ 90% | 시나리오 관찰 | 사용성 테스트 시 |
| **수정률** | AI 결과를 사용자가 수정한 비율 | ≤ 20% | 편집 로그 | 사용성 테스트 시 |

## 3. 검증 방법

### 3.1 정확도 검증 (Accuracy Test)

**방법**: 10건의 다양한 진료 기록으로 AI 구조화 실행 후,
의료 전문가 2인이 독립적으로 정확도 평가

**평가 기준**:
- ✅ Correct: 문장이 올바른 SOAP 섹션에 배치됨
- ⚠️ Acceptable: 약간 모호하나 합리적 분류
- ❌ Incorrect: 명백한 오분류

**정확도 산출**: (Correct + Acceptable) / Total × 100

**테스트 케이스 구성**:

| # | 진료과 | 복잡도 | 주요 특징 |
|---|--------|--------|-----------|
| 1 | 내과 | 중 | 만성질환 f/u, 검사 결과 다수 |
| 2 | 응급 | 고 | 급성 증상, 빠른 의사결정 |
| 3 | 소아과 | 중 | 보호자 진술, 감염 |
| 4 | 정형외과 | 저 | 단순 외상 |
| 5 | 산부인과 | 중 | 정기 검진 |
| 6 | 내과 | 고 | 다장기 문제, 영어/한국어 혼용 |
| 7 | 정신과 | 중 | 주관적 호소 비중 높음 |
| 8 | 외과 | 중 | 수술 계획 포함 |
| 9 | 내과 | 저 | 간단한 감기 |
| 10 | 응급 | 고 | 다발성 외상 |

---

### 3.2 사용성 테스트 (Usability Test)

**참가자**: 의사 또는 의료진 3~5명 (모집 가능 범위 내)
**시간**: 참가자당 20~30분

**시나리오 테스트 (Task-based)**:

| Task | 내용 | 제한 시간 |
|------|------|-----------|
| T1 | SnapSOAP에 접속하여 인터페이스를 파악하세요 | 2분 |
| T2 | 샘플 진료 기록 하나를 선택하여 구조화를 실행하세요 | 1분 |
| T3 | 결과에서 "Assessment" 섹션의 내용을 수정하세요 | 2분 |
| T4 | 직접 진료 기록을 입력하고 구조화하세요 | 5분 |
| T5 | 구조화된 결과를 복사하세요 | 1분 |

**측정 항목**:
- 각 태스크 완료 여부 (Success / Fail)
- 각 태스크 소요 시간 (초)
- 오류 발생 횟수 및 유형
- SUS 설문 (10문항, 태스크 완료 후 즉시 측정)
- 자유 피드백 (Think-aloud 프로토콜)

---

### 3.3 A/B 비교 (시간 검증)

**방법**: 동일한 진료 기록 3건에 대해

| 조건 | 설명 |
|------|------|
| (A) 수동 정리 | 참가자가 직접 텍스트를 읽고 SOAP 섹션으로 분류 |
| (B) SnapSOAP | AI 구조화 후 결과 확인 및 필요 시 수정 |

**측정**: 각 방식의 소요 시간, 최종 결과 품질(전문가 평가)

**분석**: 대응 표본 t-test 또는 Wilcoxon 검정 (n<5일 경우)

## 4. 검증 일정

| 주차 | 활동 | 산출물 |
|------|------|--------|
| Week 1 | 프로토타입 완성 + 내부 테스트 | 버그 리포트, 정확도 예비 측정 |
| Week 2 | 정확도 테스트 (10건) | 정확도 리포트 (케이스별 판정표) |
| Week 3 | 사용성 테스트 (3~5명) | SUS 점수, 태스크 완료율, Think-aloud 기록 |
| Week 4 | 결과 분석 + 개선 방향 도출 | 검증 결과 보고서 |

---

## 5. 리스크 및 대응

| 리스크 | 가능성 | 대응 전략 |
|--------|--------|-----------|
| 의료진 참가자 모집 어려움 | 중 | 최소 3명 목표, 동료/지인 네트워크 활용 |
| AI 분류 정확도 목표 미달 | 중 | 프롬프트 반복 최적화, Few-shot 예시 추가 |
| 사용성 점수 낮음 | 저 | UX 개선 우선순위 도출, 2차 테스트 실시 |
| API 응답 지연 (>5초) | 저 | 모델 변경 또는 타임아웃 UI 개선 |

---

## 6. 윤리적 고려사항

- 모든 테스트에 **더미 데이터만 사용** (실제 환자 정보 절대 사용 금지)
- 참가자에게 프로토타입임을 명확히 고지 후 동의 획득
- "AI가 의료 판단을 대체하지 않음" 사전 고지
- 테스트 데이터는 테스트 종료 후 즉시 삭제
- 참가자 발언 및 행동 기록 시 익명 처리
